# メルカリ会計資料 自動変換仕様書

## 1. 目的

メルカリからダウンロードした `sold` CSV をもとに、税理士へ提出できる会計資料を自動作成する。

単なる売上一覧ではなく、以下を説明できる資料にする。

- 商品がいつ売れたか
- 販売利益がいつメルカリ内で確定したか
- 確定した売上金が、銀行振込・メルカード支払い充当・メルペイ利用など、どこへ移動したか
- 銀行口座への入金額と、メルカリ残高の減少額が一致しているか
- メルカード請求額と銀行口座引落額に差がある場合、その差額がメルカリ残高充当で説明できるか

## 2. 前提

メルカリでは、商品購入日と売上金確定日が異なる。

- 購入日時：注文が入った日
- 購入完了日 / 更新日時：購入者評価などにより販売利益が確定した日
- 販売利益：商品代金から販売手数料・配送料・他費用を差し引いた金額

また、確定した販売利益は必ずしも銀行口座へ振り込まれるとは限らない。
メルカード支払い、メルペイ利用、その他の残高利用に充当される場合がある。

そのため、会計資料としては「売上ベース」と「売上金確定ベース」と「残高消込表」を分けて管理する。

## 3. 入力ファイル

### 3.1 メルカリ sold CSV

メルカリからダウンロードした sold CSV を入力する。

主に使用する列は以下。

- 購入完了日
- 購入日時
- 更新日時
- 取引情報URL
- 商品ID
- 商品名
- 商品URL
- 商品代金
- 販売手数料
- 配送料
- 他費用
- 税率
- 販売利益

### 3.2 メルカード情報

メルカード情報はCSVではなくスクリーンショット管理を前提とする。

初期バージョンではOCR自動読取は行わず、手入力またはCSV転記で確認する。

基本計算式：

```text
メルカード請求金額 − 銀行口座引落額 ＝ メルカリ残高充当額
```

### 3.3 銀行入金情報

銀行口座へメルカリから入金された金額を手入力またはCSV転記する。

## 4. 出力するExcelシート

- README
- sold_raw
- 売上ベース
- 売上金確定ベース
- 残高消込表
- メルカード確認
- 銀行振込確認
- 月次集計

## 5. 自動変換処理

1. sold CSV を読み込む
2. `sold_raw` に元データを保存する
3. 購入日時順に並び替えて `売上ベース` を作成する
4. 購入完了日または更新日時順に並び替えて `売上金確定ベース` を作成する
5. 売上金確定ベースの販売利益を `残高消込表` の増加として反映する
6. 残高消込表に手入力用の空行を追加する
7. メルカード確認用シートを作成する
8. 銀行振込確認用シートを作成する
9. 月次集計を作成する
10. Excel形式で出力する

## 6. 会計資料としての説明文

税理士へ説明する場合は、以下のような説明が可能。

> メルカリでは、商品が購入された日と、購入者評価等により販売利益が確定する日が異なります。  
> また、確定した売上金は必ずしも銀行口座へ全額振り込まれるわけではなく、メルカード支払いやメルペイ利用に充当される場合があります。  
> そのため、売上発生日を確認する「売上ベース」、売上金確定日を確認する「売上金確定ベース」、確定後の残高の移動を確認する「残高消込表」に分けて管理しています。
