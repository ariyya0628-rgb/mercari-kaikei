# 資料の入れ方

今後は、資料をこのプロジェクトの `input` フォルダに入れてから、次のコマンドを実行します。

```powershell
python src/main.py
```

出力先は `output` フォルダです。ファイル名は `税理士提出用_メルカリ会計資料_日時.xlsx` になります。

## 入れる資料

### sold CSV

置き場所:

```text
input/
```

または将来的な整理用:

```text
input/sold_csv/
```

ファイル名は `mercari_sold_` で始まるものを優先して読み込みます。

### メルカリ残高履歴

現在は、スクショから転記したCSVを使います。

```text
input/mercari_balance_history_screenshots_YYYYMMDD.csv
```

スクショそのものは証憑として保管してください。OCR自動読取は次の拡張候補です。

### 銀行明細

現在は、通帳や銀行PDFから転記したCSVを使います。

```text
input/yucho_bank_statement_YYYYMMDD.csv
```

PDF原本も一緒に保存しておくと、税理士さんが確認しやすくなります。

### メルカード明細

Excel化済みの明細は、クレカ明細として取り込みます。

5月・6月のように全額ポイント払いが混ざる可能性があるものは、CSV化しても `メルカード明細_未判定` に分けます。ポイント払いはクレカ仕入れに含めません。

### 領収書・レシート

まだ自動集計には入れていません。仕入れ、送料、梱包材、備品などは原本を保管し、必要に応じて別シート化します。

## 出力Excelで見る場所

- `不足資料チェック`: 何が反映済みで、何が要確認か
- `残高消込表`: メルカリ残高の増減
- `メルカード整理状況`: メルカード明細の取込状況
- `メルカード明細_未判定`: ポイント払い混在の可能性がある明細
- `銀行振込確認`: 銀行入金とメルカリ振込申請の照合
