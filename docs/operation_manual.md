# メルカリ会計資料作成ツール 操作手順書

この手順書は、このプロジェクトを作成した本人以外の人でも、目的・必要資料・操作方法・出力Excelの見方が分かるようにまとめたものです。

## 1. このツールの目的

このツールは、メルカリの売上を単純に合計するためのものではありません。

メルカリでは、次の日付や金額の動きが一致しないことがあります。

- 商品が売れた日
- 購入者評価などが完了して販売利益が確定した日
- 販売利益がメルカリ残高へ加算された日
- 残高が銀行へ振り込まれた日
- 残高がメルカード支払い・メルペイ利用などへ充当された日

このツールの目的は、税理士へ以下の流れを説明できるExcel資料を作成することです。

1. 商品が売れた
2. 購入者評価後に販売利益が確定した
3. 販売利益がメルカリ残高に加算された
4. 残高が銀行振込、メルカード支払い、メルペイ利用などで減少した
5. 最終的に残高の増減を説明できる

## 2. このツールで作成されるExcel

実行すると、`output` フォルダに次のような名前のExcelが作成されます。

```text
税理士提出用_メルカリ会計資料_YYYYMMDD_HHMMSS.xlsx
```

Excelには、主に以下のシートが作成されます。

| シート名 | 目的 |
|---|---|
| README | Excel全体の説明、税理士向けの読み方 |
| 不足資料チェック | 反映済み資料と、追加確認が必要な資料の一覧 |
| sold_raw | メルカリから出力したsold CSVの元データ |
| 売上ベース | 商品が売れた日、つまり購入日時基準の売上一覧 |
| 売上金確定ベース | 販売利益が確定した日基準の一覧 |
| 残高消込表 | メルカリ残高の増加・減少を説明する表 |
| メルカード確認 | メルカード請求額、銀行引落額、残高充当額の確認 |
| 銀行振込確認 | 銀行口座への実入金とメルカリ振込申請の照合 |
| 月次集計 | 売上ベース・売上金確定ベースの月別集計 |
| 残高履歴_スクショ転記 | メルカリ残高履歴スクショから転記した確認表 |
| 銀行明細_ゆうちょ転記 | 銀行明細から転記した確認表 |
| メルカード明細_取込 | クレカ集計対象として取り込んだメルカード明細 |
| メルカード明細_ポイント等除外 | ポイント払い・コード決済等でクレカ集計から除外した明細 |
| メルカード整理状況 | メルカード明細の月別照合結果 |

## 5. 入力資料の準備

### 5.1 メルカリ sold CSV

メルカリからダウンロードしたsold CSVを用意します。

置き場所は次のどちらかです。

```text
input/
```

または

```text
input/sold_csv/
```

ファイル名は、できれば次のように `mercari_sold_` で始めます。

```text
mercari_sold_2026_20260608_144709.csv
```

このCSVは、メルカリの売上・手数料・送料・販売利益をExcel化するための中心資料です。

### 5.2 メルカリ残高履歴

メルカリ残高履歴は、初期運用ではスクリーンショットを見ながらCSVへ転記します。

例:

```text
input/mercari_balance_history_screenshots_YYYYMMDD.csv
```

残高履歴では、以下のような動きを確認します。

- 売上金確定
- チャージ
- メルペイ清算・返済
- 振込申請
- 返金
- 事務局補填

商品売上についてはsold CSVからも取得できるため、二重計上しないように注意します。

### 5.3 銀行明細

銀行に実際に入金されたメルカリ振込を確認するため、銀行明細を用意します。

初期運用ではPDFや通帳画像を見ながらCSVへ転記します。

例:

```text
input/yucho_bank_statement_YYYYMMDD.csv
```

銀行明細では、メルカリ側の振込申請額と銀行側の実入金額の差額も確認します。差額がある場合は、振込手数料などの可能性があります。

### 5.4 メルカード明細

メルカードは、Excel化済みの明細とスクリーンショット明細が混在する想定です。

このプロジェクトでは、主に次の2種類に分けて扱います。

| 種類 | 扱い |
|---|---|
| カード払い | クレカ集計対象に入れる |
| コード決済・全額ポイント払い | クレカ集計対象から除外する |

スクリーンショット上の判定ルールは以下です。

- 商品名の下にカードのマークがあるもの: カード払い
- 商品名の下にバーコードのマークがあるもの: コード決済
- 金額の横にチェックがあるもの: 全額ポイント払い

全額ポイント払いは、実際のクレジットカード請求に含めないため、クレカ仕入れには入れません。

## 7. 出力Excelの確認方法

### 7.1 最初に見るシート

まず `不足資料チェック` を確認します。

ここで、どの資料が反映済みか、どの資料が未整理・要確認かを確認できます。

### 7.2 売上を確認する

売上の発生日を見る場合は `売上ベース` を見ます。

このシートは `購入日時` を基準にしています。

### 7.3 売上金確定を確認する

販売利益がいつ確定したかを見る場合は `売上金確定ベース` を見ます。

基準日は以下の優先順位です。

1. 購入完了日
2. 購入完了日が空欄の場合は更新日時

この基準日が、メルカリ残高へ販売利益が加算された日として扱われます。

### 7.4 残高の増減を確認する

`残高消込表` では、メルカリ残高の増減を確認します。

基本計算式は以下です。

```text
残高 = 前行残高 + 増加 - 減少
```

sold CSVから確定した販売利益は、自動的に `増加` として入ります。

銀行振込、メルカード充当、メルペイ利用などは、残高が減るため `減少` として扱います。

### 7.5 メルカードを確認する

`メルカード整理状況` を確認します。

見るべき列は以下です。

| 列 | 意味 |
|---|---|
| クレカ集計対象合計 | カード払いとして集計対象にした金額 |
| 除外合計 | ポイント払い・コード決済などで除外した金額 |
| 画面/請求合計 | スクショ画面上の合計金額 |
| 画面差額 | 明細合計と画面合計の差額 |
| 残高履歴の清算額 | メルカリ残高履歴に出てくる清算・返済額 |
| 残高差額 | 明細合計と残高履歴上の金額との差 |

`画面差額` が0であれば、スクショから転記した明細合計と画面上の合計が一致しています。

### 7.6 銀行振込を確認する

`銀行振込確認` または `銀行明細_ゆうちょ転記` を確認します。

メルカリの振込申請額と、銀行口座への実入金額が一致するか確認します。

一致しない場合は、振込手数料などの可能性を確認します。

## 8. 税理士へ説明するときのポイント

税理士へ提出する場合は、以下のように説明できます。

> メルカリでは、商品が購入された日と、購入者評価などにより販売利益が確定する日が異なります。  
> また、確定した売上金は必ずしも銀行口座へ全額振り込まれるわけではなく、メルカード支払いやメルペイ利用に充当される場合があります。  
> そのため、この資料では、売上発生日を確認する「売上ベース」、売上金確定日を確認する「売上金確定ベース」、確定後の残高移動を確認する「残高消込表」に分けて管理しています。

## 9. よくあるトラブル

### 9.1 `python src/main.py` でファイルが見つからない

PowerShellの現在位置がプロジェクトフォルダではない可能性があります。

次のように移動してから再実行してください。

```powershell
cd "C:\Users\allja\Desktop\codex-project\メルカリ売り上げ集計"
python src/main.py
```

### 9.2 `inputフォルダにメルカリsold CSVが見つかりません` と出る

sold CSVが `input` フォルダに入っていない可能性があります。

`mercari_sold_` で始まるCSVを `input` に入れてください。

### 9.3 CSVが文字化けする

このツールは以下の文字コードを順番に試します。

- UTF-8-SIG
- UTF-8
- CP932
- Shift_JIS

それでも文字化けする場合は、CSVをExcelで開いて「CSV UTF-8」形式で保存し直してください。

### 9.4 Excelが開けない

同じ名前のExcelを開いたまま再実行すると、保存できない場合があります。

Excelを閉じてから再実行してください。

### 9.5 メルカードの合計が合わない

以下を確認してください。

- スクロールで同じ行を二重転記していないか
- 取消線の行を含めていないか
- 金額横にチェックがある全額ポイント払いをクレカ集計に入れていないか
- カード払いとコード決済を混同していないか
- スクショ上部や下部で一部の行が切れていないか

## 10. GitHub運用ルール

GitHubへ上げるのは、ツール本体と説明書だけです。

上げてよいもの:

- `README.md`
- `requirements.txt`
- `src/`
- `tests/`
- `docs/`
- 空フォルダ維持用の `.gitkeep`

上げないもの:

- メルカリsold CSV
- 残高履歴CSV
- 銀行明細CSV
- メルカード明細CSV
- スクリーンショット
- 出力Excel
- PDFや領収書画像

これらは個人情報や会計情報を含むため、GitHubには上げません。

## 11. 月次作業の流れ

毎月の作業は、基本的に以下の順番です。

1. メルカリからsold CSVをダウンロードする
2. sold CSVを `input` に入れる
3. メルカリ残高履歴のスクショを保存する
4. 必要に応じて残高履歴をCSVへ転記する
5. 銀行明細を保存する
6. 必要に応じて銀行明細をCSVへ転記する
7. メルカード明細のスクショまたはCSVを保存する
8. `python src/main.py` を実行する
9. `output` のExcelを開く
10. `不足資料チェック` と `メルカード整理状況` を確認する
11. 差額があればスクショや銀行明細を見直す
12. 完成したExcelと原本資料を税理士へ提出する

このプロジェクトを別の人へ渡す場合は、以下を伝えてください。

- このツールは会計ソフトではなく、税理士へ説明するための補助資料作成ツールであること

- 最終判断は税理士に確認すること
